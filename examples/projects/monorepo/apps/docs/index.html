This example demonstrates how to use EVA CSS in a monorepo with multiple applications sharing a single configuration file.

Overview

In this monorepo setup, we have three applications:

  • Landing - Marketing landing page with hero and features
  • Dashboard - Admin dashboard with metrics and charts
  • Documentation - Technical documentation site (this page)

All three apps share the same eva.config.cjs at the monorepo root, ensuring consistent design tokens across the entire product.

Configuration

The shared eva.config.cjs contains all design tokens: 

module.exports = {
  sizes: [4, 8, 12, 16, 24, 32, 48, 64, 96, 128],
  fontSizes: [12, 14, 16, 18, 20, 24, 32, 40, 48, 64],
  buildClass: true,

  theme: {
    name: 'eva',
    lightMode: { lightness: 96.4, darkness: 6.4 },
    darkMode: { lightness: 5, darkness: 95 }
  }
};

Note: Changes to eva.config.cjs will affect all apps when you rebuild. This ensures consistency but requires coordination in team environments.

Build Commands

The monorepo package.json includes scripts for building all apps:

npm run build

Build all apps (landing, dashboard, docs)

npm run build:landing

Build only the landing app

npm run build:dashboard

Build only the dashboard app

npm run build:docs

Build only the documentation app

npm run validate

Validate the shared configuration

Key Features

Feature Description
Shared Config One eva.config.cjs for all apps
Consistent Design Same sizes, colors, theme across apps
Independent Builds Build apps separately or all at once
Auto Dark Mode OKLCH system handles theme switching

Important: In a real monorepo, you might want to use tools like Turborepo or Nx to optimize builds and caching. This example focuses on the EVA CSS configuration aspect.
